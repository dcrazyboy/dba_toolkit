## 🚀 **VSCodium : Pourquoi ce bordel ?**
Parce que :
- **C'est du VSCode sans la télémétrie** → Moins de Microsoft, plus de contrôle.
- **Tes scripts et configs sont centralisés** → Plus besoin de les recopier à chaque nouvelle install.
- **Ça gère les environnements multi-systèmes** → OpenSUSE, Rocky, Debian... ou même ton vieux PC sous Windows (on juge pas).

## 💡 **Bonnes pratiques (parce que sinon, c'est le bazar)**
- **Un `README.md` par dossier** : Si tu ajoutes un sous-dossier, **documente-le**. Sinon, dans 6 mois, même toi tu auras oublié à quoi il sert.
- **Nomme tes fichiers clairement** : `config_ultime_v3_final.sh` → **❌**. `setup_vscodium_opensuse.sh` → **✅**.
- **Teste avant de commiter** : Un script qui plante en prod, c'est comme un café renversé sur le clavier : **ça arrive, mais faut éviter**.
